About your browser...

If you ended up here, your browser is missing features needed by Bridge Designer. Sorry about that. We do our best to make Bridge Designer work with most computing devices having a mouse or similar gadget plus a broad range of up-to-date browser software.

What should work

Not phones. Sorry. We don't support pure touch devices like most smart phones and pads with no mouse.
Browsers. Up-to-date versions of these should be fine:
  • Based on recent Chromium:
    • Google Chrome
    • Microsoft Edge
    • Chromium, open source
    • Opera
  • Firefox
Other Chromium- and Firefox-based browsers may work fine. Unfortunately, we can't support Apple Safari.
Screen resolution. At least 1200 by 800 pixels. More is better.
Mouse. An accurate mouse-like pointing device. For example, a track pad or track ball should be fine.

How to fix problems

Switch browsers or upgrade. If a startup warning complained about an unsupported browser or missing HTML or CSS features, try installing the current version of one of the supported browsers. If your device is unable, try the latest possible version of Google Chrome. That's the best chance of success.
Adjust settings or hardware as needed. If a startup warning included screen resolution or mouse, you'll want to adjust or replace hardware. Pro tips:
  • Screen resolution is usually a choice. Check your operating system's display settings. You want one that's at least 1200x800.
  • For screen resolutions near the minimum, set the browser to full screen mode to use all possible pixels.
  • Many devices support adding a mouse, often for just a few dollars. Check the web.
Take your chances. Bridge Designer might be okay with the current browser despite the warnings, though some features are likely to be disabled or work poorly. You'll be notified about disabled features. Give it a try by clicking Cancel in the warning dialog.

Help us out

Report problems. Report a bug at the project home page if, after reading through all above, things still don't make sense.
Software fixes. If you're a developer, we'll happily consider your pull requests to address browser-related bugs and limitations. File a bug or feature request first to discuss what you'd like to do. See the project home page. Compatibility projects likely to be helpful:
  • Safari support.
  • Pure touch device support.
  • Handling device orientation changes.
